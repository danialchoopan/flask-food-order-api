# مستندات REST API سامانه دانیال فست فود

این سند شامل جزئیات فنی تمام نقاط انتهایی (Endpoints) سامانه برای استفاده در اپلیکیشن موبایل و وب است.

## احراز هویت (Authentication)

تمامی درخواست‌ها به جز ثبت‌نام و ورود، نیازمند توکن احراز هویت در هدر درخواست هستند.
**Header:** `Authorization: Bearer <token>`

### ۱. ثبت‌نام کاربر
- **آدرس:** `/user/register`
- **متد:** `POST`
- **بدنه درخواست (JSON):**
```json
{
  "name": "نام کاربر",
  "phone": "09120000000",
  "password": "password123",
  "city_name": "تهران"
}
```
- **پاسخ موفق (201):** `{"message": "User registered successfully"}`

### ۲. ورود کاربر
- **آدرس:** `/user/login`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "phone": "09120000000",
  "password": "password123"
}
```
- **پاسخ موفق (200):** `{"token": "access_token_here", "name": "نام کاربر"}`

## مدیریت موقعیت جغرافیایی (Location)

### ۳. بروزرسانی موقعیت جغرافیایی کاربر
- **آدرس:** `/user/update_location`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "latitude": 35.77,
  "longitude": 51.35,
  "address": "آدرس جدید کاربر"
}
```

### ۴. دریافت رستوران‌های نزدیک
- **آدرس:** `/user/restaurants/nearby`
- **متد:** `GET`
- **پارامترها:** `lat`, `lon`
- **توضیح:** لیست رستوران‌ها را بر اساس فاصله جغرافیایی برمی‌گرداند.

---

## نظرات و امتیازدهی (Ratings & Comments)

### ۵. ثبت نظر و امتیاز برای رستوران
- **آدرس:** `/add/comment`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "seller_id": 1,
  "content": "متن نظر شما",
  "rating": 5
}
```

---

## مدیریت سفارشات (Orders)

### ۶. ثبت سفارش جدید
- **آدرس:** `/add/order`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "seller_id": 1,
  "details": "[[1, 2], [2, 1]]"
}
```
*(توجه: فیلد details به صورت رشته‌ای از آرایه شامل [food_id, quantity] ارسال می‌شود)*
